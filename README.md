## 1 задание 
Вычислить и вывести на экран в виде таблицы значения функции, заданной графически, на интервале от Xнач до Xкон с шагом dx. Интервал и шаг задать таким образом, чтобы проверить все ветви программы.

## 2 задание Реализовать один тест на простоту в соответствии с индивидуальным вариантом (модуль 3).
* Варианты заданий:
1. Тест Миллера;
2. Тест Поклингтона;
3. ГОСТ Р 34.10-94.
* Входные данные: На вход программе подается целое положительное
число bits, означающее требуемую разрядность чисел, которые проверяются
тестами на простоту.
* Этапы выполнения работы:
1. Построить таблицу простых чисел, меньших 500, с помощью решета
Эратосфена.
2. Реализовать процедуру генерации простых чисел заданной
разрядности bits на основе теста на простоту.
3. Построить 10 различных чисел-кандидатов заданной разрядности
bits с помощью процедуры, разработанной на шаге 2.
4. Создать переменную rejected, означающее количество
отвергнутых чисел при проверке на простоту (только для тестов Миллера и
Поклингтона).
5. Каждое построенное число проверить тестом на простоту
следующим образом:
a. Если после проверки тестом число не является простым, то
сгенерировать новое число и увеличить значение rejected на
1;
b. Если после проверки тестом число является простым, то
перейти к шагу 6.
6. Для чисел, которые отмечены тестами как «простые», провести
дополнительную проверку этим же тестом, но при параметре надежности t,
равном 2 (только для тестов Миллера и Поклингтона). Если после
дополнительной проверки число является простым, то в качестве результата
проверки тестом на простоту вернуть true, иначе false.
7. Перейти к следующему кандидату в простые числа.
Выходные данные: Результат работы программы оформляется в виде таблицы.

## ЗАДАНИЕ 4 Увлекательная игра
Павел и Вика придумали увлекательную игру. Сначала они выписывают
в тетрадь последовательность из n целых чисел (5 ≤ n ≤ 50 000). Игроки
ходят по очереди: на каждом ходу игрок может выбрать от одного до m чисел
(2 ≤ m ≤ 100) из начала последовательности, стереть их, и прибавить сумму
стёртых чисел к своему счёту.
Вариант 2
Игрок не может стирать столько же
чисел, сколько стёр его противник
в предыдущем ходе
Павел ходит первым. Оба игрока действуют по следующей стратегии:
на каждом ходу игрок вычисляет префиксные суммы оставшихся чисел и
выбирает такое количество чисел (от 1 до m, с учётом ограничения на
повторение количества), которое даст ему максимальную сумму. Если
несколько вариантов дают одинаковую максимальную сумму, игрок выбирает
минимальное возможное количество чисел.
Игра заканчивается, когда все числа удалены из тетради, а
победителем становится тот, у кого итоговый счёт больше. Гарантируется,
что ничья невозможна.
Необходимо вывести 1 в случае победы Павла и 0 — в случае победы
Вики.
* Пример №1
* Ввод – 5 4 | 1 2 3 4 100 | Вывод - 0
* Пример №2
* Ввод – 5 4 | 1 2 3 4 -100 | Вывод - 1
* Пример №4 (вар.2)
* Ввод – 6 2 | 1 2 -3 4 -9 8 | Вывод - 0
